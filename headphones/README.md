## ğŸ› ï¸ Development Process

### 1. Planning and Wireframing
- Sketched layout sections: Hero, About, Services, Contact, Footer
- Defined content hierarchy and visual flow

### 2. HTML Structure
- Used semantic tags: `<header>`, `<main>`, `<section>`, `<article>`, `<footer>`
- Added meaningful headings and alt text for images
- Ensured logical tab order and keyboard accessibility

### 3. CSS Styling
- Applied consistent spacing, colors, and typography
- Used Flexbox and Grid for layout control
- Created reusable utility classes for margin, padding, and alignment

### 4. Accessibility Enhancements
- Used `aria-label`, `role`, and descriptive link/button text
- Ensured contrast ratios meet WCAG standards
- Enabled keyboard navigation and focus styles

### 5. Responsive Design
- Used media queries to adapt layout for mobile, tablet, and desktop
- Applied fluid widths and relative units (`em`, `%`, `vh`, `vw`)
- Tested across screen sizes and orientations
